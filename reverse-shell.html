🔍 Project Overview

This project demonstrates a fully functional reverse shell written in C, complete with persistence using Windows Registry and optional keylogging capabilities. It's intended for ethical and educational use only.

📂 Files Included

  • server.cpp – The listener that accepts commands and displays output
  • client.c – The payload that connects back to the server and executes commands

📡 How It Works

  1. The server runs and listens on a specified port (default: 50005)
  2. The client connects to the server and sends a message (simulating a compromise)
  3. The server can issue commands like:
    • cd – change directory
    • persist – add autorun entry in registry
    • keylog_start – start keylogger (threaded)
    • q – quit and close the connection

🧠 Key Features

  • ✅ Custom socket handling using Winsock2 and POSIX
  • ✅ Registry persistence using Windows API
  • ✅ Keylogger support via threads
  • ✅ Command execution with _popen()
